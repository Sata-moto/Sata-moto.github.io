Section 1:\quad Section\ 1:Section 1: 栈和队列的简单介绍

\quad\quad 相信大部分OIer，学习的第二种数据结构，不是栈，就是队列...（第一种是数组）

\quad\quad 作为OIer征战题场的先锋悍将，哪里都可以看见它们的影子...

\quad\quad 所以说啊...如果你早已洞悉它们的构造、特性与作用，那么直奔下一节就好了！不要在这一节浪费时间....

\quad\quad 1−1:1-1:1−1: 栈与队列的构造与特性

\quad\quad 以下内容截自百度百科，可能引起小白们高度舒（kong）适（huang）。

\quad\quad

\quad\quad 以下为正常内容0.0....

\quad\quad 简单而言，栈和队列是两种的线性数据结构...

\quad\quad 与数组不同，栈与队列不允许使用者对它内部元素进行修改...

\quad\quad 栈仅仅允许从“栈顶”放入元素，也只能从“栈顶”拿走元素...

\quad\quad 队列只允许从“队头”拿走元素，从“队尾”拿走元素...

\quad\quad 可以做一个形象化的比方：数组是一个大保险柜，你可以把自己想放的东西放到里面任何一个小柜子里...也可以从任何一个小柜子拿出曾放入的东西....（图片来自百度）

\quad\quad 栈是一个仅有一端开口的铁桶，你可以向它里面塞与与其等径的球，显然，你每次都只能从开口处放进去球，也只能从开口处拿出之前放进去的球...（图片来自 啊哈磊 《啊哈算法》）

\quad\quad 队列是由一群遵纪守法的好少年组成的，他们排出了长长的队伍，出来的人只能在队列头部做完自己想做的事，然后出来，进去的人只能乖乖的跑去队尾排队，然后等待...（图片来自博客园）

\quad\quad 我们不妨总结一下：

\quad\quad 数组是一种线性的，完全开放的，我们想访问哪个元素就访问哪个元素，想操作哪个元素，就操作哪个元素的数据结构...

\quad\quad 栈是一种线性的，从哪入就从哪出，内中元素无法访问的数据结构

\quad\quad 队列是一种线性的，单向入，单向出，内中元素无法访问的数据结构

\quad\quad 由此，我们可以推测它们的特性：

\quad\quad 先说栈：既然元素的出口入口是同一个，那么肯定是后放入的元素更先出来...还记得前面那个装球的桶吗？小哼（就是左边那位呆呆萌萌的家伙）一定是先放入了2球，再放入1球，再放入3球...拿出来的顺序则是先拿3球，再拿1球，最后拿2球

\quad\quad 至于队列，那就很显然了，排队的时候肯定是队首的家伙先出队嘛...先排队的人自然出来得更早...

\quad\quad 综上，我们说：栈满足“后入先出”原则，队列满足“先入先出”原则...

\quad\quad\quad\quad 1−2:1-2:1−2: queuequeuequeue,stackstackstack和dequedequedeque

\quad\quad 上一小节中，笔者描述了栈与队列的构造与特性，但就是没说它们应该怎么用代码实现...

\quad\quad 很显然，笔者在本小节里也不打算讲它们的代码实现了...他决定直接上大杀器：STL库里的 queuequeuequeue,stackstackstack和dequedequedeque

\quad\quad STL是什么？STL是指C++的标准模板库，它炒鸡好用0.0...

\quad\quad 在STL内部，良心的C++人员直接帮你打包好了栈和队列，分别用stackstackstack和queuequeuequeue表示...

\quad\quad 怎么样定义它们呢？很简单...

\quad\quad 首先，你需要两个头文件：

	#include <stack> //这是栈 
	#include <queue> //这是队列

\quad\quad 然后，你就可以像定义数组一样定义它们，不过就是定义的代码有点奇怪0.0

\quad\quad 就像这样：

	stack <int > s;
	queue <int > q;

\quad\quad 很简单地就定义了一个$int 类型的栈和一个类型的栈和一个类型的栈和一个int类型的数组，如果希望操作其他类型，只需要吧尖括号里的类型的数组，如果希望操作其他类型，只需要吧尖括号里的类型的数组，如果希望操作其他类型，只需要吧尖括号里的int$改为其他类型就好了，支持结构体，非常方便0.00.00.0...

\quad\quad 先讲一讲stackstackstack的操作：

	#include <cstdio>
	#include <stack>
	
	using namespace std;
	
	stack <int > s;
	
	int main()
	{
		int x = 10086;
	
		//向栈中放入元素（放球入栈）
		s.push(x);//等价于s.push(10086);
	
		//得到栈顶元素（不会对栈产生影响，仅仅是 得到 栈顶元素）
		int Top = s.top();
		printf("%d\n", Top); //也可直接printf("%d\n",s.top());
		//输出了10086
	
		//拿出栈顶元素（拿球出栈）
		s.pop();//弹出了10086
	
		//得到栈内元素总数
		s.push(214), s.push(213), s.push(212);
		int Size = s.size();
		printf("%d\n", Size); //也可直接printf("%d\n",s.size());
		//输出了3
	
		//判断栈是否为空
		s.pop(), s.pop(), s.pop();
		if(s.empty())
			printf("This stack is empty\n");
	
	//---------------------------------------------//
	//下面是一段演示
	
		s.push(1926),s.push(817);
		printf("%d\n",s.top());//输出817
		s.push(1926081),s.push(7);
		s.pop();//7被弹掉了...
		s.pop();//1926081被弹掉了...
		s.pop();//817被弹掉了...
		printf("%d\n",s.top());//输出1926    
		if(s.empty()) printf("This stack is empty\n");//没有反应 
		s.pop();//1926被弹掉了... 
		if(s.empty()) printf("This stack is empty\n");//输出了 
		 
		return 0;
	}

\quad\quad queuequeuequeue的操作与stackstackstack差不多:

	#include <cstdio>
	#include <queue>
	
	using namespace std;
	
	queue <int > q;
	
	int main()
	{
		int x = 10086;
	
		//向队尾中插入元素（让人入队）
		q.push(x);//等价于q.push(10086);
	
		//得到队首元素（不会对队列产生影响）
		int Front = q.front();
		printf("%d\n", Front); //也可直接printf("%d\n",q.front());
		//输出了10086
	
		//队首元素出队（让人出队）
		q.pop();//10086出队了 
	
		//得到队列内元素总数
		q.push(214), q.push(213), q.push(212);
		int Size = q.size();
		printf("%d\n", Size); //也可直接printf("%d\n",q.size());
		//输出了3
	
		//判断队列是否为空
		q.pop(), q.pop(), q.pop();
		if(q.empty())
			printf("This queue is empty\n");
	
	//---------------------------------------------//
	//下面是一段演示
	
		q.push(1926),q.push(817);
		printf("%d\n",q.front());//输出1926
		q.push(1926081),q.push(7);
		q.pop();//1926离队了...
		q.pop();//817离队了...
		q.pop();//1926081离队了...
		printf("%d\n",q.front());//输出7    
		if(q.empty()) printf("This queue is empty\n");//没有反应 
		q.pop();//7离队了... 
		if(q.empty()) printf("This queue is empty\n");//输出了 
		 
		return 0;
	}

\quad\quad 万能的STL还内置了另一个神奇的数据结构，它就是双端队列：dequedequedeque，它支持在头部插入/删除/查看，也支持在尾部插入/删除/查看...可以说，是结合了queuequeuequeue和stackstackstack的强悍数据结构（事实上，queuequeuequeue可以看做dequedequedeque头部删除，尾部插入，头部查看几个功能的结合体，stackstackstack可以看做dequedequedeque头部插入，头部删除，头部查看这几个功能的结合体）...

\quad\quad 下面给出它的主要功能：

	#include <cstdio>
	#include <deque>
	
	using namespace std;
	
	deque <int > q;
	
	int main()
	{
		q.push_back(10086);//尾插
		printf("%d\n", q.back()); //尾查
		q.pop_back();//尾删
	
		q.push_front(10086);//头插
		printf("%d\n", q.front()); //头查
		q.pop_front();//头删
	
		q.push_back(1926);
		q.push_front(817);
		printf("%d\n", (int )q.size()); //得到元素个数
		q.pop_back();
		q.pop_front();
	
		if(q.empty())//判断是否为空
			printf("This deque is empty\n");
	
		return 0;
	}

\quad\quad 2−1:2-1:2−1: 单调栈，单调队列的构造，特性与应用

\quad\quad 什么是单调栈？

\quad\quad 顾名思义，单调栈就是内部单调的栈结构...

\quad\quad 说到这，读者（肯定不）可能会感到奇怪：栈结构不是不允许访问非栈顶元素吗？我们怎么让它单调？

\quad\quad 我们，首先从一道例题说起：洛谷 P5788 【模板】单调栈

\quad\quad 粗读此题，我们可能会想到最暴力的方法：从每个元素开始，向右扫出第一个大于它的元素...复杂度O(N2)O(N^2)O(N2)...

\quad\quad 然而...

\quad\quad 显然，我们的复杂度不能超过O(NlogN)O(NlogN)O(NlogN)...

\quad\quad 有没有O(NlogN)O(NlogN)O(NlogN)的方法呢？有，但这里不提...

\quad\quad 我们思考O(N)O(N)O(N)的方法...

\quad\quad 首先，我们要注意到：本题求的是，每个元素向右第一个大于它的元素...

\quad\quad 我们能否用一个结构，把某个数右边的所有数，这一集合维护起来？

\quad\quad 显然是有的，比如说栈，从右往左依次处理每一个数，每个数处理完后往栈里一塞，对于每一个数而言，从栈顶开始找，第一个比它大的元素就是答案...

\quad\quad 然而，就算用朴素的栈维护，我们的复杂度还是O(N2)O(N^2)O(N2)，因为我们需要深入栈里去找第一个比这个数大的元素...而栈的大小均摊是O(N/2)O(N/2)O(N/2)...

\quad\quad 我们能否压缩栈的空间？使得栈中第一个比某个元素大的元素可以很快地被我们找到？我们思考：假使有一个元素A1A_1A1​，有一个元素A2A_2A2​，A1A_1A1​在A2A_2A2​的左边且A1>A2A_1>A_2A1​>A2​，那么对于在A1A_1A1​左边的数A3A_3A3​，第一个比它大的数，一定不可能是A2A_2A2​...（反证即可）

\quad\quad 这意味着什么？假如我入栈了A1A_1A1​，那么A2A_2A2​就大可以抛弃了，因为轮不到它...

\quad\quad 做一个形象化的比方：如果你旁边有人既比你小（靠左）又比你强（数值大），你就可以AFOAFOAFO了...

\quad\quad 如何具体地利用这个性质来解决本例题？

\quad\quad 假使我们正在处理A1A_1A1​元素，栈顶元素是A2A_2A2​，根据我们的朴素栈算法，我们要去栈里寻找第一个大于A1A_1A1​的元素，那么我们就应该先把A2A_2A2​拿出来，然后看看栈里下一个元素大不大于A1A_1A1​...找到元素之后，我们再把A2A_2A2​放回去，然后把A1A_1A1​放到栈顶...

\quad\quad 但是，有必要吗？为什么要把A2A_2A2​放回去？它又不可能成为A1A_1A1​右边的数的答案...

\quad\quad 所以既然我们要放入“又强又小”的A1A_1A1​，我们就干脆让A2AFOA_2AFOA2​AFO掉吧！

\quad\quad 以此类推，在栈中找到比A1A_1A1​大的元素A4A_4A4​前，栈顶所有元素都可以被丢掉...这么做大大减少了栈的大小...我们的栈的意义，从“某个数右边所有的元素”，变成了“某个数右边，所有可能对以后的答案有所贡献的元素”....

\quad\quad 以样例数据1 4 2 3 5为例...

\quad\quad 按照我们的现有算法，从右往左扫描每一个数...

\quad\quad 先扫到5，栈为空，故5的答案是0，将5放入栈中，栈内元素为 5...

\quad\quad 扫到3，栈顶元素是5，比3大，于是3的答案是5，把3放入栈中，栈内元素为 3 5...

\quad\quad 扫到2，栈顶元素是3，比2大，于是2的答案是3，把2放入栈中，栈内元素为 2 3 5...

\quad\quad 至此，我们都没有找到某个元素，既比栈中的元素靠左，又比栈中的元素大...所以栈中的元素都是有意义的元素...

\quad\quad 扫到4，栈顶元素是2，比4小...此后，2不会再对左边的任何答案产生贡献，所以我们直接抛弃2...栈顶元素更新为3，比4小，同样被抛弃...栈顶元素更新为5，比4大，所以4的答案是5，把4放入栈中，栈内元素为 4 5

\quad\quad 扫到1，栈顶元素是4，比1大，于是1的答案是4，把1放入栈中，栈内元素为 1 4 5...

\quad\quad 模拟结束...

\quad\quad 在这个过程中，我们发现，我们的栈始终保持单调递增（反证法可证）...

\quad\quad 因此，我们将这种保持单调的栈结构，叫做单调栈...

\quad\quad 算一算时间复杂度：每个点刚好都入栈一次，出栈一次，所以复杂度为稳定的O(N)O(N)O(N)...

\quad\quad 以下为代码：

#include <bits/stdc++.h>

using namespace std;

const int N = 3000010;

int n;
int a[N];

struct node
{
	//单调栈中保存每一个元素的大小与位置
	//（题目问的是右边第一个比该元素大的元素的位置）
	int wz, d;

	node (int a, int b)
	{
		wz = a, d = b;
	}
	node () { }
};

stack <node > s;

int ans[N];

//快读
int read()
{
	int x = 0, ch = '-';
	while(!(ch >= '0' && ch <= '9')) ch = getchar();
	while(ch >= '0' && ch <= '9') x *= 10, x += ch - '0', ch = getchar();
	return x;
}

int main()
{
	n = read();
	for(int k = 1; k <= n; k++)
		a[k] = read();

	//从右向左
	for(int k = n; k >= 1; k--)
	{
		//如果有的人比你强，又比你小，你就可以AFO了...
		while(!s.empty() && s.top().d <= a[k])
			s.pop();

		//更新答案，别忘了判定栈内还有没有元素
		if(!s.empty())
			ans[k] = s.top().wz;

		//把这个元素加入栈中
		s.push(node(k, a[k]));
	}

	for(int k = 1; k <= n; k++)
		printf("%d ", ans[k]);

	return 0;
}

\quad\quad 双倍经验时间到了

\quad\quad 然鹅，因为栈这个结构本身的局限性，单调栈只能解决少数最值问题...

\quad\quad 比如说这个玩意，单调栈就解决不掉：洛谷 P1886 滑动窗口 /【模板】单调队列

\quad\quad 题意很简单：求所有长度为K的区间的最大值最小值...

\quad\quad 单调栈似乎是没法求的...那么...

\quad\quad 好吧...我不说你们也知道，得用单调队列...

\quad\quad 单调队列中的“队列”，其实不是queuequeuequeue，而是万能的dequedequedeque...

\quad\quad （以下均为求最大值的情况）

\quad\quad 我们先思考最暴力的方法：裸搜...显然不行...

\quad\quad 按照上一题的思路，我们找一个结构表示滑动窗口内的元素...

\quad\quad 因为左边的元素会离开滑动窗口，所以我们使用双端队列...

\quad\quad 双端队列中维护每一次滑动后，窗口内的元素，然后我们查找答案，只需要找双端队列中的最大值就好了。

\quad\quad 如何使双端队列中的元素尽量少？还是A1A_1A1​，A2A_2A2​，A1A_1A1​在A2A_2A2​的左边且A1<A2A_1<A_2A1​<A2​，窗口从左向右滑...

\quad\quad 显然，窗口内存储A1A_1A1​是没有必要的，在A2A_2A2​离开窗口之前，A1A_1A1​没有成为最大值的机会，而A1A_1A1​肯定比A2A_2A2​先AFOAFOAFO...（杯具啊）

\quad\quad 因此，当某个元素从右边入队后，它左边的所有比它小的元素都可以AFOAFOAFO了...

\quad\quad 显然，我们又不经意间维护了队列的单调性（我为什么要加又？？？），队列必然递减（反证可得），因此，最大的元素将会是队头的元素....

\quad\quad 与上一题不同的是，本题的队头元素可能会失效（窗口不覆盖它了），因此我们要及时弹出队头元素...

\quad\quad 总结一下本题的流程：当滑动窗口右滑一格时，我们让队首的失效元素出队（队首元素可不一定失效哦）...

\quad\quad 然后把队尾那些比当前将要放进去的元素还要小的元素丢掉（又老又弱AFO）...

\quad\quad 接着放入当前多覆盖的元素...

\quad\quad 最后用当前的队首元素（最大）作为当前窗口的答案...

\quad\quad 下面是代码：

#include <bits/stdc++.h>

using namespace std;

const int N = 1000010;

int n, K;
int a[N];

struct node
{
	//这里记录每个点进入的位置
	//是因为本题中点可能会失效
	//（窗口滑走了） 
	int wz, d;

	node (int a, int b)
	{
		wz = a, d = b;
	}
	node () { }
};
deque <node > q;

int main()
{
	scanf("%d %d", &n, &K);
	for(int k = 1; k <= n; k++)
		scanf("%d", &a[k]);

	//最小值，与最大值相反 
	for(int k = 1; k <= n; k++)
	{
		//Step1 
		while(!q.empty() && q.front().wz <= k - K)
			q.pop_front();

		//Step2 
		while(!q.empty() && q.back().d >= a[k])
			q.pop_back();

		//Step3
		q.push_back(node(k, a[k]));

		//Step4
		if(k >= K)
			printf("%d ", q.front().d);
	}
	printf("\n");

	while(!q.empty()) q.pop_back();

	//最大值 
	for(int k = 1; k <= n; k++)
	{
		//Step1 
		while(!q.empty() && q.front().wz <= k - K)
			q.pop_front();

		//Step2 
		while(!q.empty() && q.back().d <= a[k])
			q.pop_back();

		//Step3
		q.push_back(node(k, a[k]));

		//Step4
		if(k >= K)
			printf("%d ", q.front().d);
	}

	return 0;
}

#include <bits/stdc++.h>

using namespace std;

int pla[70];//某种颜色的球最晚出现在哪

struct node
{
	int wz, col;

	node (int a, int b)
	{
		wz = a, col = b;
	}
	node () { }
};
deque <node >q;

int tot, answ = 0x3f3f3f3f;

int main()
{
	scanf("%d %d", &n, &K);
	for(int k = 1; k <= K; k++)
	{
		int num;
		scanf("%d", &num);

		for(int i = 1; i <= num; i++)
		{
			int a;
			scanf("%d", &a);

			v.push_back(Node(k, a));
		}
	}

	sort(v.begin(), v.end());

	memset(pla, -1, sizeof(pla));

	for(int k = 0; k < n; k++)
	{
		//滚出去
		if(pla[v[k].col] == -1) tot++;
		pla[v[k].col] = v[k].wz;
		
		//拿进来
		q.push_back(node(v[k].wz, v[k].col));

		//不能要
		while(!q.empty() && q.front().wz != pla[q.front().col])
			q.pop_front();

		//求最值
		if(tot == K) answ = min(answ, v[k].wz - q.front().wz);
	}

}

\quad\quad 2−2:2-2:2−2: 单调队列优化DP

\quad\quad 单调队列除了本身可以解决某些最值问题外，还可以用来优化DP...

\quad\quad 举个栗子：洛谷 P1725 琪露诺

\quad\quad 看起来像一道很简单的DP题？（其实就是一道很简单的DP题）

\quad\quad 我们不妨令f[i]f[i]f[i]表示从起点跳到第i个点所获得的最大冰冻指数...

\quad\quad 显然，初态有f[0]=0f[0]=0f[0]=0，其余f[i]=−INFf[i]=-INFf[i]=−INF...

\quad\quad 末态则取能跳到nnn以后的点f[i]f[i]f[i]的最大值...

\quad\quad 转移方程：f[i]=max(f[i−j])+A[i],j∈[L,R]f[i]=max(f[i-j])+A[i],j∈[L,R]f[i]=max(f[i−j])+A[i],j∈[L,R]

\quad\quad 如何转移？朴素的转移最坏复杂度可以达到O(N2)O(N^2)O(N2)，显然不行..

\quad\quad 我们注意到，转移的过程本质上就是求区间最大值的过程...

\quad\quad 如果我们要求f[i]f[i]f[i]，我们就要求出f[i−R]−−f[i−L]f[i-R] -- f[i-L]f[i−R]−−f[i−L]区间内的最大值...

\quad\quad 随着我们的iii的递增，我们要得到的区间两端标号$i-R $ 和 i−Li-Li−L同样递增...

\quad\quad 想到什么了？对！滑动窗口！

\quad\quad 这里本质上不就是求某一个大区间内(0−N)(0-N)(0−N)中，每一个长度为R−LR-LR−L的小区间的最大值吗？

\quad\quad 套用滑动窗口的模板，我们就可以在O(N)O(N)O(N)的时间里完成转移...

\quad\quad 代码如下（里面有一些小技巧值得注意）：

#include <bits/stdc++.h>

using namespace std;

const int N = 200010;

int n, L, R;
int a[N], f[N];

struct node
{
	int wz, d;
	node (int aa, int bb)
	{
		wz = aa, d = bb;
	}
	node () { }
};
deque <node > q;

int answ = 0xefefefef;

int main()
{
	memset(f, 0xef, sizeof(f));

	scanf("%d %d %d", &n, &L, &R);
	for(int k = 0; k <= n; k++)
		scanf("%d", &a[k]);

	f[0] = 0;

	for(int k = 0; k <= n; k++)
	{
		//step 1
		while(!q.empty() && q.front().wz < k - (R - L))
			q.pop_front();

		//step 2
		while(!q.empty() && q.back().d <= f[k])
			q.pop_back();

		//step 3 
		q.push_back(node(k, f[k]));

		//step 4 
		if(k + L <= n) f[k + L] = q.front().d + a[k + L];
		else answ = max(answ, q.front().d);
	}

	printf("%d", answ);

	return 0;
}

\quad\quad 一般而言，在DP过程中，如果我们要求出某写线性连续区间内的最大值，单调队列维护是一个不错的选择，它可以将原本O(N2)O(N^2)O(N2)的内部循环复杂度降至O(N3)O(N^3)O(N3)...

\quad\quad 再看一道练习：洛谷 P2254 [NOI2005]瑰丽华尔兹

\quad\quad （题面太长，就不截图了）...

\quad\quad 看到题目后，我们进行分析：

\quad\quad 首先，这题是个DPDPDP，时间、坐标都是很显然的维度...

\quad\quad 最裸的方法，无非是设f[k][i][j]f[k][i][j]f[k][i][j]表示在k时间结束后，来到了(i,j)(i,j)(i,j)处所滑动的最大距离...

\quad\quad 转移很显然，初末状态也很显然...（如果你这都不觉得显然...那么还是别往下看了吧）

\quad\quad 当然，时间复杂度也很显然：O(TN2)O(TN^2)O(TN2)，肯定是有的MLE,有的点TLE...

\quad\quad 思考优化：因为在每一个时间段内，船倾斜的方向是一样的...所以，自上个时间段结束后，点(x,y)(x,y)(x,y)一定是顺着船倾斜的方向，移动0−−t0--t0−−t个单位（ttt表示该时间段长度）...

\quad\quad 所以我们尝试设f[k][i][j]f[k][i][j]f[k][i][j]表示在k时间段结束后，来到了(i,j)(i,j)(i,j)处所滑动的最大距离（一字之差，差了两百倍的复杂度）...

\quad\quad 初态：f[0][ori_x][ori_y]f[0][ori\_x][ori\_y]f[0][ori_x][ori_y]（(ori_x,ori_y)(ori\_x,ori\_y)(ori_x,ori_y)为起点坐标）

\quad\quad 末态：f[K][i][j]f[K][i][j]f[K][i][j]中最大的那个...

\quad\quad 转移：f[k][i][j]=max(f[k−1][i′][j′]+dist[(i′,j′),(i,j)])f[k][i][j]=max(f[k-1][i'][j']+dist[(i',j'),(i,j)])f[k][i][j]=max(f[k−1][i′][j′]+dist[(i′,j′),(i,j)])

\quad\quad 其中，(i′,j′)(i',j')(i′,j′)为可以转移过来的点的坐标...

\quad\quad 大体上不难解决...复杂度O(KN2)O(KN^2)O(KN2)，绝（开）对（O）不（2）能（就）AC...

\quad\quad 这里给出这样优化后的代码：

#include <bits/stdc++.h>

using namespace std;

const int N = 210;

//偏移数组
int py[4][2] = { 1, 0, -1, 0, 0, 1, 0, -1};

int n, m, ori_x, ori_y, K;

char ch[N][N];

//时间段
struct node
{
	int L, R, to;
} Times[N];

//在第k个时间段结束后，在坐标为(x,y)的点的最长滑行路径
int f[N][N][N];

int main()
{
	scanf("%d %d %d %d %d", &n, &m, &ori_x, &ori_y, &K);
	for(int k = 1; k <= n; k++)
		scanf("\n%s", &ch[k][1]);
	for(int k = 1; k <= K; k++)
		scanf("%d %d %d", &Times[k].L, &Times[k].R, &Times[k].to);

	//初始化
	memset(f, 0xef, sizeof(f));
	f[0][ori_x][ori_y] = 0;

	//DP
	for(int k = 1; k <= K; k++)
		for(int i = 1; i <= n; i++)
			for(int j = 1; j <= m; j++)
				if(ch[i][j] == '.')
					for(int move = 0; move <= Times[k].R - Times[k].L + 1; move++)
					{
						int new_x = i + py[Times[k].to - 1][0] * move;
						int new_y = j + py[Times[k].to - 1][1] * move;

						if(new_x > n || new_y > m || new_x < 1 || new_y < 1) break;
						if(ch[new_x][new_y] == 'x') break;

						if(f[k - 1][new_x][new_y] >= 0)
							f[k][i][j] = max(f[k][i][j], f[k - 1][new_x][new_y] + move);
					}

	int answ = 0xefefefef;

	for(int k = 1; k <= n; k++)
		for(int i = 1; i <= m; i++)
			answ = max(answ, f[K][k][i]);

	printf("%d", answ);

	return 0;
}

\quad\quad 例题：洛谷 P2219 [HAOI2007]修筑绿化带

\quad\quad 几乎是模板题...公园相当于大矩形，A*B是黄色矩形，花坛是红色矩形，绿化带是黄色部分...

\quad\quad 唯一的坑点在于：要求绿化带严格包围花坛，边界不允许重合...

\quad\quad 解决办法相当简单：直接把A,B各自－2，然后在最后统计的时候稍作修改即可...

\quad\quad 放出代码：

#include <bits/stdc++.h>

using namespace std;

const int MAXN = 1010;

int M, N, A, B, C, D;
int d[MAXN][MAXN];//原始矩阵
int cl[MAXN][MAXN];//前缀和，用来求“面积 ”

int Min[MAXN][MAXN];//以i,j为右下角的绿地中花坛的最小面积
int Mid[MAXN][MAXN];//上面数组的中途数组，右下角的点在  第i行的 宽度为绿地宽度 的区域内的 花坛的最小面积

struct node
{
	int wz, d;

	node (int a, int b)
	{
		wz = a, d = b;
	}
	node () { }
};

//单调队列
deque <node > q;

int answ;

//给出右下角点的坐标和竖直方向及水平方向距离，计算面积的函数
int get_S(int x, int y, int len1, int len2)
{
	return cl[x][y] - cl[x - len1][y] - cl[x][y - len2] + cl[x - len1][y - len2];
}

int main()
{
	memset(Mid, 0x7f, sizeof(Mid));
	memset(Min, 0x7f, sizeof(Min));

	//M*N 的矩阵？ 这变量名真是让人...Orz
	scanf("%d %d %d %d %d %d", &M, &N, &A, &B, &C, &D);
	for(int k = 1; k <= M; k++)
		for(int i = 1; i <= N; i++)
			scanf("%d", &d[k][i]), cl[k][i] = d[k][i] + cl[k - 1][i] + cl[k][i - 1] - cl[k - 1][i - 1];

	A -= 2, B -= 2;

	//先横着来一遍
	for(int k = C; k <= M; k++)
	{
		while(!q.empty())
			q.pop_back();

		for(int i = D; i <= N; i++)
		{
			//step 1
			while(!q.empty() && q.front().wz - D < i - B)
				q.pop_front();

			//step 2
			while(!q.empty() && q.back().d >= get_S(k, i, C, D))
				q.pop_back();

			//step 3
			q.push_back(node(i, get_S(k, i, C, D)));

			//step 4
			if(i >= B)
				Mid[k][i] = min(Mid[k][i], q.front().d);
		}
	}

	//再竖着来一遍
	for(int k = B; k <= N; k++)
	{
		while(!q.empty())
			q.pop_back();

		for(int i = C; i <= M; i++)
		{
			//step 1
			while(!q.empty() && q.front().wz - C < i - A)
				q.pop_front();

			//step 2
			while(!q.empty() && q.back().d >= Mid[i][k])
				q.pop_back();

			//step 3
			q.push_back(node(i, Mid[i][k]));

			//step 4
			if(i >= A)
				Min[i][k] = min(Min[i][k], q.front().d);
		}
	}

	int answ = 0;

	A += 2, B += 2;

	//最后统计... 
	for(int k = A; k <= M; k++)
		for(int i = B; i <= N; i++)
			answ = max(answ, get_S(k, i, A, B) - Min[k - 1][i - 1]);

	printf("%d", answ);

	return 0;
}

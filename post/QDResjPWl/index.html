Lucas定理学习笔记

发布于 2020-02-28 | 分类于 数学 、 知识点 | 3分钟 | 592字数

Lucas这个名字貌似挺常见的来着？

前言：

\quad\quad 今天早上莫名其妙的WA了一道题...

\quad\quad 怎么调都搞不定...

\quad\quad 翻了翻题解发现，原来有个叫Lucas定理的玩意...

\quad\quad 记下来以供以后参考...

正文：

\quad\quad 首先由一个问题引入：如何求 Cmn mod pC_m^{n}\bmod{p}Cmn​modp 的值？

\quad\quad 看上去很简单，把式子展开为 m!n!∗(m−n)!\frac{m!}{n!*(m-n)!}n!∗(m−n)!m!​，求阶乘求逆元然后乘起来就好。

\quad\quad 但是如果 p<mp<mp<m 呢？此时 m!≡0(modp)m! \equiv 0 \pmod pm!≡0(modp)，边做边模显然会输出0(如果 p<=np<=np<=n 那么该式子甚至无意义)。

\quad\quad 这显然并不一定是正确结果，比如说：C61≡2(mod4)C_6^{1} \equiv 2 \pmod 4C61​≡2(mod4)。

\quad\quad Lucas定理正是用来解决这类问题的。

\quad\quad 直接上它的表示公式：Cmn≡Cm/pn/p∗Cm mod pn mod p(modp)C_m^{n} \equiv C_{m/p}^{n/p} *C_{m \bmod p}^{n \bmod p} \pmod pCmn​≡Cm/pn/p​∗Cmmodpnmodp​(modp)。

\quad\quad 很显然这是一个递归的过程。

\quad\quad 均摊下来，它可以将上界为 $ m$ 的组合数计算转化为上界为 min⁡(m,p)\min(m,p)min(m,p) 的组合数计算。

\quad\quad 证明这里略去，推荐一篇csdn博文。

\quad\quad 下面给出 Lucas 定理的例题和代码。

\quad\quad 例题：洛谷P3807 【模板】卢卡斯定理。

\quad\quad 代码：

  #include <bits/stdc++.h>
  #define int long long

  using namespace std;

  const int N = 2e5 + 10;

  inline int read()
  {
      int x = 0, ch = '~';

      while(!(ch >= '0' && ch <= '9')) ch = getchar();
      while(ch >= '0' && ch <= '9') x = (x << 3) + (x << 1), x += ch - '0', ch = getchar();

      return x;
  }

  int n, m, p;
  int fac[N], inv[N];

  void init()
  {
      n = read(), m = read(), p = read();
      fac[0] = fac[1] = inv[0] = inv[1] = 1;
      for(register int k = 2; k < p; k++)
          fac[k] = fac[k - 1] * k % p;
      for(register int k = 2; k < p; k++)
          inv[k] = (p - p / k) * inv[p % k] % p;
  }

  int C(int x, int y)
  {
      if(y < x) return 0;
      return (fac[y] * inv[fac[x]] % p) * inv[fac[y - x]] % p;
  }

  int Lucas(int x, int y)
  {
      if(!x) return 1;
      return Lucas(x / p, y / p) * C(x % p, y % p) % p;
  }

  signed main()
  {
      int opt = read();
      while(opt--)
          init(), printf("%lld\n", Lucas(m, n + m));

      return 0;
  }

结语：

\quad\quad Lucas定理还是比较简单的，柿子记住了就没啥问题。

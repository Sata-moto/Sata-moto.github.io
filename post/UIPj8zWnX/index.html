三分法学习笔记

发布于 2020-02-28 | 分类于 数学 、 知识点 | 6分钟 | 1050字数

既然有二分法，那为什么不能三分呢？

前言：

\quad\quad 又是一个小知识点来着...

\quad\quad 日常做笔记。

正文：

\quad\quad 三分法，是一种(在摄影、绘画、设计等艺术中经常使用的构图手段)求凸函数极值的方法。

\quad\quad 在了解它之前，我们先要了解什么是凸函数。

	凸函数是一个定义在某个向量空间的凸子集 C 上的实值函数 f。
	而且对于凸子集 C 中任意两个向量 x1、x2，有 f((x1+x2)/2)<=(f(x1)+f(x2))/2 成立。
																						-------百度百科

\quad\quad 通俗的说，就是长得和二次函数差不多的函数，比如说 $f(x)= |x^3| $。

\quad\quad 三分法就是用来计算这种函数的极值的方法。

\quad\quad 其思路是：首先给定初始区间[l,r][l,r][l,r]，保证极值对应的 xxx 在[l,r][l,r][l,r]区间内，然后于[l,r][l,r][l,r]中任取两个点 a,ba,ba,b，这里假设 a<ba<ba<b。此时，区间被分成了三段：[l,a],[a,b],[b,r][l,a],[a,b],[b,r][l,a],[a,b],[b,r]，根据 a,ba,ba,b 点的大小关系，可以舍去 [l,a][l,a][l,a] 区间或者舍去 [b,r][b,r][b,r] 区间，使得问题规模缩小。重复处理直到区间左右端点足够接近。

\quad\quad 显然，三分法最关键的部分就在于 “根据 a,ba,ba,b 点的大小关系，可以舍去 [l,a][l,a][l,a] 区间或者舍去 [b,r][b,r][b,r] 区间” 这一操作。这个操作对于不同的函数，不同的大小关系来讲，分别不同，这里就不举栗子了。

\quad\quad 如果我们要让三分尽量快，我们就要让每次舍去的最多，显然，当 aaa 从左趋近于 l+r2\frac{l+r}{2}2l+r​，bbb 从右趋近于 l+r2\frac{l+r}{2}2l+r​时，每次删去的区间长度趋近于 l+r2\frac{l+r}{2}2l+r​，最大。因此，我们每一次三分时，可以直接在中点附近取点（而不是分别在两个三等分点位置取点），以获得接近二分的复杂度。

\quad\quad 模板：P3382 【模板】三分法（虽然给出的函数不一定是凸函数，但题目保证了给出区间内的子函数是凸函数）

\quad\quad 代码：

#include <bits/stdc++.h>

using namespace std;

const int N = 16;
const double MIN = 0.00000001;

int n;
double l, r, d[N];

//计算函数值
double F(double x)
{
	double answ = d[n];
	for(register int k = n; k >= 1; k--)
		answ *= x, answ += d[k - 1];

	return answ;
}

signed main()
{
	scanf("%d %lf %lf", &n, &l, &r);
	for(register int k = 0; k <= n; k++)
		scanf("%lf", &d[n - k]);

	//三分
	while(r - l > MIN)
	{
		double mid = (l + r) / 2;
		if(F(mid - MIN) < F(mid + MIN))
			l = mid;
		else
			r = mid;
	}

	printf("%.5lf", l);

	return 0;
}

\quad\quad 练习：P2571 [SCOI2010]传送带，枚举传送带上走到的两个点，三分套三分。

\quad\quad 代码：

#include <bits/stdc++.h>

using namespace std;

const double Minn = 0.00001;

int Ax, Ay, Bx, By, Cx, Cy, Dx, Dy, P, Q, R;

double dist(double x1, double y1, double x2, double y2)
{
	return sqrt(pow(x1 - x2, 2) + pow(y1 - y2, 2));
}

inline int read()
{
	int x = 0, ch = '~';

	while(!isdigit(ch)) ch = getchar();
	while(isdigit(ch)) x = (x << 3) + (x << 1), x += ch - '0', ch = getchar();

	return x;
}

double f(double x, double x0, double y0, double len1)
{
	double x1 = (Dx - Cx) * x + Cx, y1 = (Dy - Cy) * x + Cy;
	double len2 = dist(x0, y0, x1, y1), len3 = dist(x1, y1, Dx, Dy);
	return len1 / P + len2 / R + len3 / Q;
}

double F(double x)
{
	double x0 = (Bx - Ax) * x + Ax, y0 = (By - Ay) * x + Ay;
	double len1 = dist(Ax, Ay, x0, y0), l = 0, r = 1;

	while(r - l > Minn)
	{
		double mid = (l + r) / 2;

		if(f(mid - Minn, x0, y0, len1) > f(mid + Minn, x0, y0, len1))
			l = mid;
		else
			r = mid;
	}

	return f(l, x0, y0, len1);
}

signed main()
{
	Ax = read(), Ay = read(), Bx = read(), By = read();
	Cx = read(), Cy = read(), Dx = read(), Dy = read();
	P = read(), Q = read(), R = read();

	double l = 0, r = 1;
	while(r - l > Minn)
	{
		double mid = (l + r) / 2;

		if(F(mid - Minn) > F(mid + Minn))
			l = mid;
		else
			r = mid;
	}

	printf("%.2lf", F(l));

	return 0;
}

结语：

\quad\quad 总有人把三分和求导二分弄混，它们的实现显然不同好吗。

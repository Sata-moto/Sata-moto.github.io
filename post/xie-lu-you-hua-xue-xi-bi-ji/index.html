N方 D P 要A掉

斜率优化少不了

前言：

\quad\quad 曾经看过斜率优化...当时没有看懂Orz...

\quad\quad 因为CSPCSPCSP也不用...就没怎么在意...

\quad\quad 今天花了半个下午，A掉了斜率优化的板子...写一篇学习笔记加以巩固...

正文：

\quad\quad 先上板子：洛谷 P3195 [HNOI2008]玩具装箱TOY /【模板】斜率优化

\quad\quad 题目大意是：把nnn个物品分成若干组，如果某组中放有多个物品，每两个物品间要空一个单位长度，物品的长度为CiC_iCi​，每一组带来的代价是。该组长度lll与某一固定常数LLL之差的平方，求最小代价和...

\quad\quad 本题DPDPDP方程是非常显然的（如果你觉得不显然，那么重修DPDPDP去吧）...

\quad\quad 令f[k]f[k]f[k]表示，把前面kkk个物品分好组后的最小代价...

\quad\quad 则有：

f[k]=min(f[i]+(cl[k]−cl[i]+k−i−1−L)2);f[k] = min(f[i] + (cl[k] - cl[i] + k - i - 1 - L)^2); f[k]=min(f[i]+(cl[k]−cl[i]+k−i−1−L)2);

\quad\quad 显然，转移内部的iii需要从000穷举到k−1k-1k−1，复杂度为O(n2)O(n^2)O(n2)...

\quad\quad DPDPDP部分代码：

for(int k = 1; k <= n; k++)
	for(int i = 0; i < k; i++)
		f[k] = min(f[k], f[i] + (int )pow((cl[k] - cl[i] + k - i - 1 - L, 2));
//cl为C的前缀和

\quad\quad 作为一道紫题，本题自然不会让人就这样轻而易举的A掉，它的数据范围为：

1<=n<=5∗1041<=n<=5*10^4 1<=n<=5∗104

\quad\quad 如何优化本题的时间复杂度？外层的111~nnn的循环大概是优化不得，所以只能从里面的循环下手...

\quad\quad 有没有觉得DP方程里的min(f[i]+(cl[k]−cl[i]+k−i−1−L)2)min(f[i] + (cl[k] - cl[i] + k - i - 1 - L)^2)min(f[i]+(cl[k]−cl[i]+k−i−1−L)2)很熟悉？

\quad\quad 还记得单调队列优化针对的DPDPDP式子吗？也是min(f[i]+XXX)min(f[i] + XXX)min(f[i]+XXX)，不过，这里显然不能使用单调队列优化，因为(cl[k]−cl[i]+k−i−1−L)2)(cl[k] - cl[i] + k - i - 1 - L)^2)(cl[k]−cl[i]+k−i−1−L)2)会随着kkk的改变而改变，所以根本无法直接放入单调队列...

\quad\quad 这个时候，就该本文的主角：斜率优化出场了...

\quad\quad 斜率优化，顾名思义，是通过斜率对DPDPDP进行优化...它优化的并非是状态转移方程，而是像单调队列那样，优化状态转移方式...

\quad\quad 让我们拿出我们的DPDPDP方程，做一些小小的变形...

f[k]=min(f[i]+(cl[k]−cl[i]+k−i−1−L)2)f[k] = min(f[i] + (cl[k] - cl[i] + k - i - 1 - L)^2) f[k]=min(f[i]+(cl[k]−cl[i]+k−i−1−L)2)

\quad\quad 括号里的式子很烦人不是？我们尝试把括号拆开...

\quad\quad 硬拆括号得算死人...我们稍微换元一下：

\quad\quad 令A[k]=cl[k]+k,B[i]=cl[i]+i+1+LA[k]=cl[k]+k,B[i]=cl[i]+i+1+LA[k]=cl[k]+k,B[i]=cl[i]+i+1+L...

\quad\quad 仅仅只是为了好算，换元了一下而已，如果你喜欢，也可以吧B[i]B[i]B[i]里面的常数丢A[k]A[k]A[k]里面去...（注意，换元后的A[k],B[i]A[k],B[i]A[k],B[i]与之前的cl[k],cl[i]cl[k],cl[i]cl[k],cl[i]在性质上无本质区别）
\quad\quad 式子变为：

f[k]=min(f[i]+(A[k]−B[i])2);f[k] = min(f[i] + (A[k]-B[i])^2); f[k]=min(f[i]+(A[k]−B[i])2);

\quad\quad 展开...

f[k]=min(f[i]+A[k]2+B[i]2−2∗A[k]∗B[i])f[k] = min(f[i] + A[k]^2+B[i]^2-2*A[k]*B[i]) f[k]=min(f[i]+A[k]2+B[i]2−2∗A[k]∗B[i])

\quad\quad 我们看看式子中的哪些东西是会变的...

\quad\quad 首先，所有与kkk有关的数值都可以视作不变，因为我们转移求f[k]f[k]f[k]时，是对于某一个特定的kkk求的...其他与iii有关的式子都会随iii改变...

\quad\quad 我们尝试把minminmin拿掉，反正这个minminmin，等价于取一个iii，让f[k]f[k]f[k]最小，我们只要在最后处理就可以了...

f[k]=f[i]+A[k]2+B[i]2−2∗A[k]∗B[i]f[k] = f[i] + A[k]^2+B[i]^2-2*A[k]*B[i] f[k]=f[i]+A[k]2+B[i]2−2∗A[k]∗B[i]

\quad\quad 我们稍微移一下项（不要问为什么这么移...反正这么移就可以）... −−−−−-----−−−−−①

B[i]2+f[i]=2∗A[k]∗B[i]+f[k]−A[k]2B[i]^2+f[i]=2*A[k]*B[i]+f[k]-A[k]^2 B[i]2+f[i]=2∗A[k]∗B[i]+f[k]−A[k]2

\quad\quad 我们注意到：等式左边的项，会随着iii的改变而改变，等式右边的B[i]B[i]B[i]也会这么改变...其余的地方，都是常数...

\quad\quad 我们不妨把B[i]B[i]B[i]看做自变量(X)(X)(X)，B[i]2+f[i]B[i]^2+f[i]B[i]2+f[i]看做自变量(Y)(Y)(Y)，认为是B[i]B[i]B[i]的改变造成了B[i]2+f[i]B[i]^2+f[i]B[i]2+f[i]的改变...变成这样：

Y=2∗A[k]∗X+f[k]−A[k]2Y=2*A[k]*X+f[k]-A[k]^2 Y=2∗A[k]∗X+f[k]−A[k]2

\quad\quad 观察这个方程，因为剩下的都是些常数，所以它本质上是一个一次函数...我们可以视2∗A[k]2*A[k]2∗A[k]为斜率(K)(K)(K)，f[k]−A[k]2f[k]-A[k]^2f[k]−A[k]2为yyy轴上截距(B)(B)(B)...

Y=KX+BY=KX+B Y=KX+B

\quad\quad （从①开始的移项是有通式的：把既与kkk有关，又与iii有关的项合并在一起，看做KXKXKX，把只与iii有关的项看做YYY，其他的项看做BBB）

\quad\quad 因为对于每一个iii而言，在计算f[k]f[k]f[k]的时候，B[i],B[i]2+f[i]B[i],B[i]^2+f[i]B[i],B[i]2+f[i]，都是定值...所以计算f[k]f[k]f[k]前已经计算过的点，可以看做坐标系中的一个个定点...就像这样：

\quad\quad 而我们现在的目的是让求出f[k]f[k]f[k]最小时的iii，所以本质上就是求一条斜率恒定（K=2∗A[k]K=2*A[k]K=2∗A[k]为定值）的直线，过某个点集中的哪个点，可以获得最小截距（B=f[k]−A[k]2,BB=f[k]-A[k]^2,BB=f[k]−A[k]2,B越小，f[k]f[k]f[k]越小）...

\quad\quad 我们做出那条斜率恒定的直线：

\quad\quad 让它上移，直到与某个节点相交，那时，它的截距最小，有f[k]=Y−KX+A[k]2f[k]=Y-KX+A[k]^2f[k]=Y−KX+A[k]2...如下图：

\quad\quad 然而，怎么求直线与点群的第一个交点（从下往上平移的第一个交点，即）呢？

\quad\quad 首先，我们得看出来：点群中的某些点是完全没有必要存在的...

\quad\quad 比如说F,E,B,IF,E,B,IF,E,B,I点，无论直线怎么移动，只要是让截距最小，就一定轮不到它们第一个相交...

\quad\quad 所以我们大可以把它们删掉...

\quad\quad 话说你有没有找到这几个点的共同点？

\quad\quad 看看下图：

\quad\quad 这里把“最为靠近”直线的点连成了一个凸包，显然，这个凸包内的点就是一定不可能与直线第一个相交的点...

\quad\quad 所以，本题中，我们只需要维护凸包就好...

\quad\quad 而本题中的X(B[i])X(B[i])X(B[i])显然是递增的，所以，可以用单调队列维护凸包，如下图：

\quad\quad 原本A,B,C,DA,B,C,DA,B,C,D构成了凸包，插入了节点EEE以后，凸包被打破...

\quad\quad 因为，组成凸包的线段的斜率，应该从左往右递增，所以我们可以从被打破的点（DDD）开始，向左删节点，直到凸包重新出现...

\quad\quad 本图中，C，EC，EC，E连成的边，与A,BA,BA,B构成了新的凸包...

\quad\quad 而对于凸包而言，什么时候第一次与一条直线相交？

\quad\quad 显然是凸包与直线lll相切与某个点的时候...

\quad\quad 根据凸包的性质，相切的点左边的，构成凸包的所有直线一定斜率小于lll，右边的直线则一定大于lll...

\quad\quad 本题中又很特殊：直线的斜率KKK，递增...

\quad\quad 所以，我们可以判断凸包最左侧的直线斜率是否大于直线lll，如果大于，那么切点一定是直线该直线左端点，否则该直线可以删去（以后的l′l'l′斜率一定大于lll，斜率不会小于该直线）...

\quad\quad 这其实是单调队列的原理...所以我们可以用单调队列维护凸包包括的直线的斜率（等价于维护凸包上的点，我更喜欢维护点...）

\quad\quad 上面讲的很乱，不过大致把本题斜率优化的操作流程讲了一遍，下面总结具体步骤：

1.求出前缀和之类必要的玩意...写清楚换元函数，不然自己都看不懂自己在写什么...

2.依次算出每一个k的DP值...这里又分几个小步骤
   
    首先，把单调队列左边的，以后永远用不上的点删掉
    
    然后，取队首节点为直线与凸包切点，算出直线方程，求得f[k]
    
    接着，在加入f[k]对应的节点(x,y)前，删掉单调队列右边的，在加入（x,y）后会破坏凸包的点
    
    最后，加入点（x,y）...
    
3.输出f[n]


\quad\quad 步骤的理由和具体操作，上面都有，自己去找...

\quad\quad 对了，注意初始时，要把f[0]=0f[0]=0f[0]=0对应的节点放到单调队列里面去...

\quad\quad 最后...放代码：

#include <bits/stdc++.h>
#define int long long
//不开long long 见祖宗

using namespace std;

const int N = 50010;

int n, L , C[N] , cl[N] , f[N];

int head = 1, tail = 1, x[N], y[N];

int A(int k)
{
	return cl[k] + k;
}
int B(int k)
{
	return cl[k] + k + 1 + L;
}
  
//算斜率，不开double都没脸见祖宗
double slope(int a, int b)
{
	return (y[a] - y[b]) / (1.0 * x[a] - x[b]);
}
double slope(int x1, int y1, int x2, int y2)
{
	return (y1 - y2) / (1.0 * x1 - x2);
}

signed main()
{
	scanf("%lld %lld", &n, &L);
	for(int k = 1; k <= n; k++)
		scanf("%lld", &C[k]), cl[k] = cl[k - 1] + C[k];

	x[tail] = B(0), y[tail++] = B(0) * B(0);

	for(int k = 1; k <= n; k++)
	{
		//进格
		while(head + 1 < tail && slope(head, head + 1) < 2 * A(k))
			head++;

		//计算
		f[k] = y[head] - 2 * A(k) * x[head] + A(k) * A(k);
		int new_x = B(k), new_y = f[k] + B(k) * B(k);

		//退格
		while(head + 1 < tail && slope(x[tail - 1], y[tail - 1], new_x, new_y) < slope(tail - 1, tail - 2))
			tail--;

		//填入
		x[tail] = new_x, y[tail++] = new_y;
	}

	printf("%lld", f[n]);

	return 0;
}

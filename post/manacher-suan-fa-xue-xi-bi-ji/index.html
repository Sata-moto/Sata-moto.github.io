manacher算法学习笔记

发布于 2020-02-04 | 分类于 字符串 、 知识点 | 8分钟 | 1716字数

前言：

\quad\quad 我一向炒鸡反感字符串算法来着...

\quad\quad 不过在某位dalaodalaodalao说：manachermanachermanacher炒鸡简单之后...

\quad\quad 我忍不住看了看manachermanachermanacher...发现它真的是炒鸡简单啊！

\quad\quad 十分钟就可以看懂的那种！

正文：

\quad\quad Emm...manacherEmm...manacherEmm...manacher（马拉车）算法是啥？

\quad\quad 顾名思义，它就是一个外国人带着一匹马和一辆车创造的的算法...

\quad\quad 用来解决一个字符串中，最长回文串的长度，这一经典问题...

\quad\quad 例如：洛谷 P3805 【模板】manacher算法

\quad\quad 题意：给定一个字符串，求其最长回文子串长度...

\quad\quad 解决这种问题...最暴力的方法是什么？

\quad\quad 显然，O(N2)O(N^2)O(N2)枚举字符串子串，再加上O(N)O(N)O(N)判断是否回文...

\quad\quad 这样显然会T爆掉的...思考更优秀的方法：

\quad\quad 观察发现，回文串一定是关于某一个字符（以下称该字符为回文中心）对称的串...

\quad\quad 其实并不一定...偶数长度的回文串并不符合上述条件...

\quad\quad 我们不妨做一个转化：我们在原字符串中的各个字符间（包括两个顶点处），插入某一个题目中不出现的字符...比如说‘#’...

\quad\quad 举个例子：有原串aabcb...转化后变为#a#a#b#c#b#...

\quad\quad 此时我们发现，不管是原串中长度为奇数的串，还是长度为偶数的串，在新串中都出现了回文中心...其中原偶数串aa(现#a#a#)对应回文中心为‘#’,原奇数串bcb（现#b#c#b#），对应回文中心为‘c’...

\quad\quad 而此时，回文串的长度恰好为原回文串的两倍+1...

\quad\quad 给出转化部分代码：

	n = strlen(&ch[1]);
	for(int k = 1; k <= n; k++)
		s[k * 2] = ch[k], s[k * 2 - 1] = '#';
	n *= 2, s[++n] = '#';

\quad\quad 这样，我们很容易思考出第二种暴力：O(N)O(N)O(N)枚举回文中心，O(N)O(N)O(N)把回文串往两边拓展...直到拓展至最大...

\quad\quad 我们称回文半径r[i]r[i]r[i]为以iii为回文中心的回文串，扩展到边界[l,r][l,r][l,r]（l,rl,rl,r关于iii对称）时，从iii到边界的距离（包括i与边界节点）...

\quad\quad 比如说，aba的以b为回文中心的回文半径为2...

\quad\quad 显然，一个回文串的边界为i−r[i]+1,i+r[i]−1i-r[i]+1,i+r[i]-1i−r[i]+1,i+r[i]−1，原始长度为r[i]−1r[i]-1r[i]−1

\quad\quad 我们可以得到第二种暴力的代码(仅仅两行诶)（我代码里循环变量常用k...不要在意）：

	for(int k = 1; k <= n; answ = max(answ, r[k]), k++)
		for(r[k] = 1; k + r[k] <= n && k - r[k] >= 1 && s[k + r[k]] == s[k - r[k]]; r[k]++);

\quad\quad 然鹅，上述代码也T了一个点...

\quad\quad 实际上，因为在无法扩展时，上述算法会跳出第二层循环，所以其对于随机数据的复杂度并不比manachermanachermanacher劣多少...

\quad\quad 好吧...我偏题了...我们考虑如何把上述算法再进行优化...

\quad\quad 可以观察：我们的第一层循环是从左往右的...这意味着，当我们在计算r[i]r[i]r[i]时，对于所有jjj满足j<i,r[j]j<i,r[j]j<i,r[j]都是已经被计算出来的...

\quad\quad 一个很简单的优化思路是：利用对称性，增大r[i]r[i]r[i]初始值...不让它从1开始搜...

\quad\quad 什么意思？

\quad\quad 上图中，我们已经计算到了iii点（为了好看，我把图中所有字母大写了），这表明我们已经计算过了jjj点的回文半径，图中[l,r][l,r][l,r]表示jjj点的边界（即从l−rl-rl−r的串是以jjj为回文中心的回文串），那么，我们一定可以找到iii关于jjj对应的i′i'i′点（实际上就是2∗j−i2*j-i2∗j−i点)...

\quad\quad 下面，我们分两种情况讨论：

\quad\quad 第一种情况，以i′i'i′点为回文中心的回文串的左边界>=L>=L>=L，我们可以发现，此时，r[i]>=r[i′]r[i]>=r[i']r[i]>=r[i′]，所以r[i]r[i]r[i]初始值可以直接设定为r[i′]r[i']r[i′]...然后开始拓展（理由很简单，对称性嘛...其实在以i′i'i′点为回文中心的回文串的左边界>L>L>L时，r[i]==r[i′]r[i]==r[i']r[i]==r[i′]，我们不拓展也是可以的...）

\quad\quad 第二种情况，以i′i'i′点为回文中心的回文串的左边界<L<L<L，此时，iii点为回文中心的回文串的右边界>=R>=R>=R，所以初始值可以直接设定为R−i+1R-i+1R−i+1...

\quad\quad 好了...我们发现，在找到了一个合适的JJJ点的情况下，我们可以快速的求出r[i]r[i]r[i]...

\quad\quad 现在最后的问题就是：如何找到JJJ点...

\quad\quad 其实不用找的，我们只需要记录在搜过的所有点中，RRR最靠右的节点，把它当做JJJ点就好...

\quad\quad 其实这样子的JJJ点，并不一定是让r[i]r[i]r[i]初始值最大的JJJ点...不过这样很方便...

\quad\quad 另外，当找不到这样的JJJ点的时候（也就是说，曾经扫过的所有点中，R最大的点的R任然小于i）

\quad\quad 我们就直接让r[i]r[i]r[i]从1开始就好...

\quad\quad 为了让变量名更清晰，我们把扫过的节点中RRR最大的点的RRR称做maxrmax_rmaxr​，那个点本身称作midmidmid...

\quad\quad 然后就有了如下代码...

#include <bits/stdc++.h>

using namespace std;

const int N = 30000000;

char ch[N], s[N];

int n, r[N], max_r, mid, answ;

void manacher()
{
	for(int k = 1; k <= n; k++)
	{
		if(k <= max_r) r[k] = min(r[mid * 2 - k], max_r - k + 1);
		else r[k] = 1;

		for(; k - r[k] >= 1 && k + r[k] <= n && s[k - r[k]] == s[k + r[k]] ; r[k]++);

		answ = max(answ, r[k]), max_r = max(max_r, k + r[k] - 1), mid = max_r == k + r[k] - 1 ? k : mid;
	}

	printf("%d", answ - 1);
}

int main()
{
	scanf("%s", &ch[1]), n = strlen(&ch[1]);
	for(int k = 1; k <= n; k++)
		s[k * 2] = ch[k], s[k * 2 - 1] = '#';
	n *= 2, s[++n] = '#';

	manacher();

	return 0;
}

\quad\quad manacher还是挺简单的字符串算法了...
